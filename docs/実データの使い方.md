# 実データの使い方

このチュートリアルは既定で合成データを使い、鍵が無くても完全オフラインで動く。ここでは
本物の市場データを取得し、実データでカーブを構築してリスクを測るまでの導線を示す。合成
データで一通り学んだあと、同じコードを実データに差し替える体験をするための手順。

## 1. FRED API キーを取得する

FRED（米セントルイス連銀）の無料 API キーを取得する。

1. https://fred.stlouisfed.org/docs/api/api_key.html でアカウント作成・キー発行（無料）。
2. リポジトリ直下の `.env` にキーを書く（`.env` は `.gitignore` 対象）。

```bash
cp .env.example .env
# .env を編集:  FRED_API_KEY=あなたのキー
```

キーはコード・notebook の出力に絶対に書かない。`bondlab.data` が `.env` から読む。

## 2. 実データを取得する（S0-2）

`bondlab.data.fred_series` は取得結果を `data/cache/`（Git 管理外）に保存し、次回以降は
ネットワーク無しで同じ結果を返す。

```python
from bondlab import data
us10 = data.fred_series("DGS10")   # 米国債10年 CMT 利回り（%）
us2  = data.fred_series("DGS2")    # 2年
print(us10.tail())
```

`nb_0002_市場データ取得基盤` の `USE_LIVE` は、`.env` に鍵があり
`BONDLAB_DATA_MODE != "cache_only"` のとき自動で True になる。鍵を入れて notebook を
開き直せば、合成サンプルの代わりに実データが流れる。

主な FRED 系列: `DGS1 DGS2 DGS3 DGS5 DGS7 DGS10 DGS20 DGS30`（UST CMT）、
`SOFR`（翌日物 RFR）、`DFII10`（10年実質金利）、`BAMLC0A0CM`（IG 社債 OAS）。

## 3. 実データでカーブを構築する（S2-1）

取得したパー利回りを年次グリッドに整えてブートストラップする。`bootstrap_par` は等間隔
テナー前提なので、CMT の年限を年次グリッドへ補間してから渡す。

```python
import numpy as np
from bondlab import data
from bondlab.curve import bootstrap_par

ids   = ["DGS1","DGS2","DGS3","DGS5","DGS7","DGS10","DGS20","DGS30"]
tenors= np.array([1,2,3,5,7,10,20,30.0])
par   = np.array([float(data.fred_series(i).dropna().iloc[-1]) for i in ids]) / 100.0

grid  = np.arange(1, 31.0)                 # 年次グリッド
par_g = np.interp(grid, tenors, par)
curve = bootstrap_par(grid, par_g, frequency=1)
print("10y ゼロレート:", round(curve.zero_rate(10.0), 5))
```

以降は合成データのときと同じ。`bondlab.analytics` でデュレーション・KRD、`bondlab.pricing`
でスワップ・オプション、`bondlab.bt` で戦略、と同じコードがそのまま実データに乗る。

## 4. 日本の銘柄別データ（JSDA）

銘柄別の rich/cheap 分析（S9）には、日本証券業協会（JSDA）の売買参考統計値（全銘柄・
日次・無料）が一次ソースになる。財務省の国債金利情報は年限グリッドのパー利回りのみで、
銘柄別ではない点に注意。いずれも取得ダウンローダは `bondlab.data` に同じ流儀で足せる。

## 5. 再配布はしない

取得した生データはリポジトリにコミットしない（`data/cache/` は `.gitignore` 対象）。
FRED の ICE BofA 指数系列や JSDA データは第三者ライセンスや利用規約があるため、加工物・
生データの再配布は避ける。教材に載せるのは合成データと取得コードのみ。
