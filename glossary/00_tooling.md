# 用語集 — 環境・データ基盤（S0）

金融の分野別用語は `01`〜`10` の各ファイルに置く。ここには開発環境と
データ取得にまつわる用語をまとめる。

<a id="virtual-environment"></a>
### 仮想環境（virtual environment）
- 定義: プロジェクト単位で Python の依存パッケージを隔離する実行環境。
  `python -m venv .venv` で作る。
- 関連: パッケージ
- 初出: S0-1

<a id="package"></a>
### パッケージ（package）
- 定義: 再利用可能なモジュールの集合。本シリーズでは自作ライブラリ `bondlab`
  を指す。`data/curve/models/...` のレイヤーに分ける。
- 初出: S0-1

<a id="unit-test"></a>
### ユニットテスト（unit test）
- 定義: 関数やクラス単位で、入力に対する期待挙動を自動検証する仕組み。
  `pytest` で実行する。ライブラリ変更時の回帰を防ぐ。
- 関連: 継続的インテグレーション
- 初出: S0-1

<a id="ci-continuous-integration"></a>
### 継続的インテグレーション（CI, continuous integration）
- 定義: コミット時に自動でテスト・実行チェックを走らせる仕組み。本シリーズ
  では鍵の無い環境で notebook が動くことをキャッシュ専用モードで確認する。
- 関連: ユニットテスト, API キー
- 初出: S0-1

<a id="discount-factor"></a>
### 割引係数（discount factor）
- 定義: 将来の1単位キャッシュフローの現在価値。レート $r$、年数 $t$、連続複利で
  $DF = e^{-rt}$。$m$ 回複利で $DF = (1 + r/m)^{-mt}$。
- 数式: $DF(t) = e^{-r t}$（連続複利）
- 関連: 複利規約
- 初出: S0-1（本格的な扱いは S1-1）

<a id="compounding-convention"></a>
### 複利規約（compounding convention）
- 定義: 利息を年に何回複利計算するかの取り決め。年1回・半年・四半期・月次・
  連続。同じ割引係数でも規約が違えば表示レートが変わる。
- 数式: 連続複利への変換 $r_c = m \ln(1 + r_m/m)$
- 関連: 割引係数
- 初出: S0-1（本格的な扱いは S1-1）

<a id="par-yield"></a>
### パー利回り（par yield）
- 定義: その年限の標準的なクーポン債が額面（パー）で取引されるときの利回り。
  年限ごとに1点で、カーブ構築の入力になる。割引に直接使うゼロレートとは別物。
- 関連: CMT, 売買参考統計値
- 初出: S0-2（剥ぎ取りは S2-1）

<a id="constant-maturity-treasury"></a>
### CMT（constant maturity treasury）
- 定義: 一定年限に補間した米国債利回り。FRED の DGS 系列（DGS10 は10年）。
  実在銘柄そのものではなく、年限を固定した補間値。
- 関連: パー利回り
- 初出: S0-2

<a id="jsda-reference-prices"></a>
### 売買参考統計値（JSDA reference prices）
- 定義: 日本証券業協会が公表する、個別銘柄ごとの平均値単価・利回り。日次・
  全銘柄・無料。銘柄別の相対価値分析（S9）の一次データ。
- 関連: ベンチマーク銘柄
- 初出: S0-2

<a id="benchmark-issue"></a>
### ベンチマーク銘柄（benchmark issue）
- 定義: 各年限を代表する銘柄。直近に発行され流動性が高い銘柄（on-the-run）が
  なることが多い。
- 関連: 売買参考統計値
- 初出: S0-2（rich/cheap は S9-1）

<a id="api-key"></a>
### API キー（API key）
- 定義: 外部サービスの認証に使う秘密情報。本シリーズでは FRED のキー。`.env`
  に置き `.gitignore` で除外し、コード・出力セルに書かない。
- 関連: 継続的インテグレーション
- 初出: S0-2

<a id="regression-test"></a>
### 回帰テスト（regression test）
- 定義: 過去に修正したバグの再発を検知するため、既知の正しい挙動（数値・
  QuantLib 突合結果）を固定して継続的に検証するテスト。仕様を確認する単体
  テストと対で、変更の安全網になる。
- 関連: ユニットテスト, カバレッジ
- 初出: S11-1

<a id="coverage"></a>
### カバレッジ（coverage）
- 定義: テストがコードのどれだけの行・分岐を実行したかの割合。高いほど未検証の
  経路が少ない目安になるが、100% でも正しさを保証はしない（何を検証するかが本質）。
- 関連: 回帰テスト
- 初出: S11-1

<a id="api-design"></a>
### API設計（API design）
- 定義: 公開する関数・クラスの名前・引数・返り値を、使いやすく壊れにくく定める
  設計。bondlab では層ごとに責務を分け、下位層にだけ依存させることで、変更の
  影響範囲を局所化する。
- 関連: セマンティックバージョニング, パッケージ
- 初出: S11-1

<a id="semantic-versioning"></a>
### セマンティックバージョニング（semantic versioning）
- 定義: バージョンを MAJOR.MINOR.PATCH の三つ組で表し、MAJOR=後方互換を壊す変更、
  MINOR=後方互換な機能追加、PATCH=バグ修正、と意味を約束する規約。利用者は更新の
  安全性を番号から読める。`0.x` 系は初期開発段階で MINOR でも破壊的変更が許容される。
- 数式: なし
- 関連: API設計, パッケージ
- 初出: S11-1
