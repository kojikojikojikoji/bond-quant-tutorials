# S1 債券の基礎

## シリーズの概要

このシリーズは、債券とデリバティブの評価すべてが乗る「一番下の土台」を固めます。将来のキャッシュフローをいくらに割り引くか（複利規約）、2つの日付の間が何年ぶんか（日数計算規約）、価格と利回りをどう往復するか（価格式とYTM）、市場で表示する価格と実際に払う金額をどうつなぐか（クリーン/ダーティ価格）——この4点を、数式の導出・スクラッチ実装・QuantLib との突合まで一貫して扱います。

以降のシリーズ（カーブ構築・リスク・デリバティブ・相対価値）は、すべてここで作る部品の上に成り立ちます。土台の規約を1つ取り違えると誤差が全ポジションに波及するため、最初に厳密に揃えることを重視します。

## このシリーズで学ぶこと

- 単利・複利・連続複利の3系統で割引を書き分け、規約間の変換を往復誤差ゼロで再現する
- 割引係数（discount factor）とゼロレートの相互変換を、規約を明示して実行する
- 主要な日数計算規約4種（ACT/360・ACT/365F・30/360・ACT/ACT）の定義と、どの市場で使うかの対応
- 営業日調整（翌営業日・修正翌営業日・前営業日）つきのクーポン日スケジュール生成
- 固定利付債の価格式を割引の合計として導出し、YTM の定義・前提・限界を説明する
- Newton法・Brent法による YTM の逆算と、その収束条件・失敗ケース
- 経過利子の規約別導出と、クリーン価格⇔ダーティ価格の相互変換
- 各国国債（UST・JGB・Gilt・Bund）の決済慣行・利回り慣行の違いと、ACT/ACT の ISDA / ISMA 変種の使い分け

## 実務での使われ方（ファンドはどう稼ぐか）

ここで扱う技術は、債券デスク・ファンドの収益とリスク管理の全経路に効きます。

- **マーケットメイク（スプレッド収益）**: セルサイドのトレーダーは買値と売値の差で稼ぎます。引き合い（RFQ）は利回りで来て約定は価格で行うため、価格⇔利回りの高速な往復が両面クォートの前提です。スプレッドは数bpしかなく、日数計算や経過利子を取り違えると利益が計算ミスで消えます。
- **相対価値（RV）**: 割安なキャッシュフローを買い割高を売る戦略は、YTM を共通言語に銘柄間の rich/cheap を測ります。国ごとに違う規約（UST は ACT/ACT、JGB は ACT/365F 単利、マネーマーケットは ACT/360）を正しく揃えて初めて、実在する歪みと見かけの差を区別できます。
- **キャリー戦略**: 保有で積み上がる経過利子とレポ金利の差でキャリーを稼ぎます。経過利子は受渡日基準で決まるため、決済慣行（T+1 / T+2）を1日ずらすと利ざやの符号すら狂います。
- **リスク管理・ALM**: デュレーションやコンベキシティは価格-利回り曲線の性質から出ます。凸性を理解しないまま利回りだけで割安判断すると、価格変動の非対称性で想定外の損を出します。

土台の規約が合っていることは、これらすべての稼ぎ方の前提条件です。

## 前提知識

- 高校〜大学初級の数学（等比数列、指数・対数、極限、Newton法の考え方）
- Python の基本（`numpy`、日付操作）
- 債券の基本用語（額面・クーポン・満期・利回り）は各 notebook 内で定義するため、事前知識は最小限で足ります

## 使う bondlab モジュール

| モジュール | 役割 |
|---|---|
| `bondlab.rates` | 複利規約の変換、割引係数とゼロレートの相互変換 |
| `bondlab.daycount` | 日数計算規約（ACT/360・ACT/365F・30/360・ACT/ACT）による年数計算 |
| `bondlab.bond` | 固定利付債の価格・YTM・経過利子（`FixedRateBond.accrued` 等） |

## 検証方針

- スクラッチ実装・`bondlab`・QuantLib の3経路が、全規約で機械精度で一致することを確認します。
- 日数計算とスケジュールは、QuantLib の `DayCounter` / `Schedule` を基準に100ケース規模で突合します。
- 複利規約の変換は往復（順変換→逆変換）で誤差ゼロを再現します。
- 経過利子は `bondlab.bond.FixedRateBond.accrued` と QuantLib の両方に突合します。

すべての実行は、リポジトリルートで `PYTHONPATH=$PWD` を付けて行ってください。
