# S2 イールドカーブ構築

## シリーズ概要

このシリーズは、市場で観測できる利回り（パー利回り・スワップレート・先物）から、
評価に使えるゼロカーブ（割引係数の期間構造）を組み立てる技術を扱います。ゼロカーブは
債券・スワップ・先物・オプションのどれを評価するにも最初に必要になる共通の入力で、
以降のシリーズ（リスク計測・デリバティブ評価・相対価値）はすべてこのカーブの上に
積み上がります。ブートストラップの理論から始め、補間・パラメトリックフィット・
マルチカーブと進み、最後に毎日壊れずに回す運用パイプラインまでを一気通貫で作ります。

## 学ぶこと

- ゼロレート・フォワードレート・パーレートが同じ割引係数の別表現であること、
  パー債の価格1条件を連立方程式として解くブートストラップ
- 「何を補間するか」（割引係数・その対数・ゼロレート・フォワード）の選択が
  フォワードの形に与える影響と、単調凸法を含む4手法の比較
- Nelson-Siegel / Svensson による少数因子（レベル・スロープ・曲率）へのカーブ圧縮と、
  安定なフィットのための2段階推定
- LIBOR改革後のマルチカーブ（OIS割引・推計カーブの分離）とデュアルブートストラップ、
  SOFR先物の凸性調整
- 入力検証・外れ値検出・前日比較・フォールバックを備えた日次カーブ構築パイプライン

## 実務での使われ方（ファンドの稼ぎ方の観点）

ゼロカーブは、値付け・裁定・リスク管理のすべての起点です。マーケットメイクのデスクは
カーブから理論値を引き、そこにビッド・オファースプレッドを乗せて稼ぎます。相対価値
ファンドは、フィットしたカーブと各銘柄の実際の利回りの差（rich/cheap）を測り、割安を買い
割高を売って乖離の平均回帰で利ざやを取ります。キャリー戦略では、保有債の利息とレポ
調達コストの差を期間ごとに評価するためにフォワードと割引が要ります。マクロのデスクは
Nelson-Siegel の因子時系列を金融政策や景気局面と重ねて方向観を作ります。

カーブの品質は、そのまま収益とリスクの信頼性を決めます。補間の癖はフォワードのノイズと
なってヘッジ比率や rich/cheap 判定を狂わせ、割引カーブの取り違えは担保付スワップの
NPVを系統的にずらします。日次パイプラインが壊れれば、汚れたカーブがファンド全体の
マーク・P&L・リスク量に波及します。だからこそカーブ構築はフロントのクオンツとリスク
管理が担う中核業務であり、金利デスク・運用会社・RVファンドのいずれを志望する場合でも
土台になります。各 notebook 冒頭の「実務での位置づけ」で、テーマごとの稼ぎ方との
繋がりを整理しています。

## 使う bondlab モジュール

- `bondlab.curve.DiscountCurve` — 割引係数の期間構造と補間の共通インターフェース
- `bondlab.curve.bootstrap_par` — パー利回りからのブートストラップ
- `bondlab.curve.nss` / `bondlab.curve.fit_nss` — Nelson-Siegel / Svensson の評価とフィット

## 検証方針

- ブートストラップは、構築したカーブでパー債を再評価して価格1（パー再現）を確認する
- Nelson-Siegel / Svensson は、既知パラメータから生成した曲線をフィットで復元する
  解析的な round-trip で検証する
- 補間は QuantLib（`DiscountCurve` / `PiecewiseLogLinearDiscount`）や scipy と突合し、
  時間軸（year fraction）を揃えたうえで一致、または想定どおりの差を数値で確かめる
- マルチカーブは、シングルカーブ評価との同一スワップのNPV差を数値で提示する
- 合成サンプルは `data/samples/synthetic_ust_par_curve.csv` と `..._par_panel.csv`
