# S3 リスク指標

## シリーズ概要

債券ポジションが金利変化でいくら損益を出すかを測り、束ね、最悪の場合を見積もる技術を扱うシリーズです。1本の債券の金利感応度（デュレーション・DV01）から始め、それをテナー別に分解し（キーレートデュレーション）、カーブ全体が確率的に動く世界（シナリオ・PCA）へ広げ、最後にポートフォリオの損失を確率で語る指標（VaR・ES）まで、リスク計測を一段ずつ積み上げます。各指標は定義から手で実装し、`bondlab` の実装や QuantLib と突合して数値の正しさを確認します。

## 学ぶこと

- マコーレー・修正・実効デュレーションとコンベクシティの違い、テイラー展開による価格変化の2次近似、負のコンベクシティが生じる仕組み
- DV01・BPV・PV01 の定義と使い分け、ポートフォリオDV01の集計、複数銘柄でのDV01ニュートラルなヘッジ比率、非平行シフトで残るPnLの要因分解
- キーレートデュレーション（パーシャルDV01）によるテナー別リスク分解、KRD合計と全体DV01の整合、バーベル／ブレットの比較
- カーブ変動の4類型（パラレル・スティープ・フラット・バタフライ）と、PCAによるレベル・スロープ・曲率への分解、ストレスシナリオの生成と再評価
- VaR と Expected Shortfall の3方式（ヒストリカル・パラメトリック・モンテカルロ）、ESの劣加法性、Kupiec 検定によるバックテスト

## 実務での使われ方（ファンドの稼ぎ方・リスク管理の観点）

このシリーズの指標は、主にリスク管理クオンツ・モデルバリデーション・マクロ／レラティブバリューのリスク管理で使われます。ファンドの稼ぎとの繋がりは次の通りです。

- **リスク量の配分と収益の綱引き**: DV01リミット・VaR／ESリミットは各デスクが取れるリスク量の上限であり、レバレッジの限度を決めます。リスクを正しく測れて初めて、収益機会にどこまで資本を張ってよいかを判断できます。
- **アルファの純度を守るヘッジ**: レラティブバリューは割安・割高の収束（キャリー）で稼ぎますが、金利感応度を打ち消して「価格のゆがみ」だけを残さなければ、狙ったアルファに金利方向の賭けが混入します。DV01・KRDはそのヘッジ数量に直結します。
- **カーブ取引の設計**: スティープナー・フラットナー・バタフライは、KRDプロファイルとPCAのスロープ・曲率因子で組み立てます。テナー別・因子別の感応度が見えないと、どのカーブ形状に賭けているかを制御できません。
- **最悪シナリオの事前把握と資本計算**: ストレステストとES（FRTBが採用）は、通常のリミット内に見えるポジションが極端な変動でいくら失うか、所要自己資本がいくらかを事前に見積もります。過大な推計は資本を無駄にし、過小な推計は突然の大損失で資本を毀損します。

（フレームワークの全体像は `docs/債券ファンドの業務.md` を参照。本シリーズはレート側リスク管理 S3 に対応します。）

## 使う bondlab モジュール

- `bondlab.analytics`
  - `duration_convexity` — 固定利付債のデュレーション・コンベクシティ・DV01 を解析的に計算
  - `effective_duration` — 価格関数の実効デュレーションを中心差分で計算
  - `bump_curve` — カーブの特定テナー近傍のゼロレートをバンプ（KRD・シナリオ用）
  - `pca` — カーブ変動行列（行=日, 列=テナー）の主成分分析
- `bondlab.risk`
  - `historical_var` / `historical_es` — 経験分布によるVaR・ES
  - `parametric_var` / `parametric_es` — 正規分布仮定によるVaR・ES
  - `kupiec_pof` — Kupiec の POF（proportion of failures）検定
- 補助として `bondlab.bond`・`bondlab.curve`（債券・割引カーブの生成）を利用します。

## 検証方針

- **デュレーション・コンベクシティ・DV01**: 解析式・バンプ法（数値微分）・QuantLib の三者を機械精度で突合する。
- **KRD**: テナー別のパーシャルDV01の合計が、全体（並行バンプ）のDV01・デュレーションと整合することを数値で確認する。
- **DV01ヘッジ**: 解析DV01・バンプ再評価DV01・修正デュレーション由来DV01の一致を確認し、バンプ幅・差分方式の誤差を定量化する。
- **PCA**: 上位主成分がレベル・スロープ・曲率に対応することと、寄与率（累積分散説明率）を確認する。
- **VaR・ES**: 手実装と `bondlab.risk` の一致を確認し、劣加法性の反例を構成、Kupiec 検定で被覆率をバックテストする。
