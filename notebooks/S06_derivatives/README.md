# S6 金利デリバティブ

## シリーズ概要

金利スワップから始めて、キャップ／フロア、スワップション、コーラブル債、そして
ボラティリティサーフェス（SABR）までを一気通貫で扱うシリーズです。S2 で構築した
イールドカーブと、S4-5 のニュメレール変換、S5-3 の Hull-White ツリーを土台にして、
金利オプションの評価・キャリブレーション・リスク計測を、スクラッチ実装と QuantLib
突合の両輪で身につけます。線形商品（スワップ）→ 標準オプション（キャップ・
スワップション）→ 経路依存・早期行使（コーラブル）→ ボラの面の記述（SABR）という
順で、難度と実務的な重みが上がっていきます。

## 学ぶこと

- FRA の複製論法から無裁定フォワードレートを導き、スワップを固定債と変動債の交換として
  評価する。パースワップレート・アニュイティ・マルチカーブ・DV01 を自作する。
- キャップをキャップレットに分解し、Black'76 をフォワード測度で正当化して値付けする。
  フラットボラとスポットボラの剥ぎ取り、対数正規ボラと正規（Bachelier）ボラの相互変換。
- スワップションをフォワードスワップレートのオプションとしてアニュイティ測度で評価する。
  Jamshidian 分解の成立条件、キャッシュ／物理決済の差、ストラドルのベガ・ガンマ。
- コーラブル債を「ストレート債 − コールオプション」に分解し、Hull-White 三項ツリー上の
  後退帰納で評価する。OAS ソルバー、Zスプレッドとの差、負のコンベクシティ、行使境界。
- スワップションのボラティリティキューブと SABR の4パラメータの役割。Hagan 近似の実装・
  フィット・近似が壊れる領域の把握。

## 実務での使われ方

これらは銀行レートデスクのオプション帳を値付けし、そのリスクをヘッジするための
道具です。スワップはマーケットメイクのスプレッド収益と相対価値（カーブ取引・
スワップスプレッド）の中核であり、キャップ・スワップションはボラティリティを売買して
インプライドと実現の差から収益を得る商品です。コーラブル債・OAS は、オプション対価
だけ上乗せされた利回りを取りにいく投資判断（OAS 投資）と、負のコンベクシティの動的
ヘッジの題材になります。SABR はこれら全オプションを整合的に値付けするための面の
記述で、スキュー自体の相対価値取引の入力でもあります。各 notebook 冒頭の「実務での
位置づけ」で、収益源との対応（`docs/債券ファンドの業務.md`）を具体化しています。

## 使う bondlab モジュール

- `bondlab.pricing` — `par_swap_rate` / `swap_annuity`（スワップ）、`black76` / `bachelier` /
  `implied_vol_black` / `caplet`（キャップ・フロア）、`swaption_black`（スワップション）、
  `sabr_vol`（SABR スマイル）
- `bondlab.curve` — `DiscountCurve` / `bootstrap_par`（割引カーブ・ブートストラップ）
- `bondlab.models` — `HullWhite`（コーラブル債評価のツリー）

## 検証方針

線形・標準オプション（スワップ・Black'76・Bachelier・スワップション・SABR）は QuantLib
（`VanillaSwap` / `blackFormula` / `bachelierBlackFormula` / `CapFloor` 等）と機械精度で
突合します。コーラブル債は、自作の Hull-White 三項ツリーによる後退帰納の評価を QuantLib
`CallableFixedRateBond` と数セント以内で突合し、格子の刻みに依存する誤差の範囲で一致を
確認します。スクラッチ実装・`bondlab`・QuantLib の三者が揃うことで、値付けの正しさと
再現性（モデルバリデーションの観点）を担保します。
