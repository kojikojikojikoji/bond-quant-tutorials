# S9 相対価値・トレーディング

## シリーズ概要

国債デスクの相対価値（RV）トレーディングを、シグナルの定義からバックテストによる評価まで一気通貫で扱うシリーズです。方向性の金利観に賭けるのではなく、証券間・カーブ上の価格の歪みが収束（または発散）する方に賭けて稼ぐ、ヘッジファンドの主戦場を対象にします。rich/cheap の平均回帰、カーブ曲率のバタフライ、キャリーとロールダウン、先物と現物の裁定という4つの収益源を順に実装し、最後にそれらを先読みのないバックテスト基盤の上で評価します。各手法がなぜ収益を生むかは `docs/債券ファンドの業務.md` の相対価値・キャリーの枠組みに沿って各 notebook 冒頭で解説します。

## 学ぶこと

- フィット済みカーブからの残差として rich/cheap を定義し、Z スコア化して銘柄横断で比較する分析基盤をスクラッチで組む
- AR(1) から平均回帰半減期を換算し、戻る歪みと戻らない構造プレミアム（on-the-run/off-the-run）を切り分ける
- DV01 中立のバタフライを 50-50・リスクウェイト・PCA の3方式で構築し、PnL をキャリー・ロール・カーブ変化・コンベクシティへ要因分解する
- レポ・ファンディングを差し引いたキャリーの厳密版とロールダウンを定義し、「キャリー＋ロール ≒ スポット − フォワード」を数値検証する
- コンバージョンファクター・グロス／ネットベーシス・インプライドレポを計算し、CTD の特定と金利シフトによる CTD スイッチを分析する
- ルックアヘッドのないバックテスターを組み、シャープレシオ・最大ドローダウン・回転率とコスト感応度で戦略を評価する

## 実務での使われ方

RV ファンドは、これらのシグナルで見つけた数 bp の歪みを、レポでレバレッジをかけて多数積むことで意味のある収益に育てます。cheap 銘柄を買い rich 銘柄を売って収束を待つ間、金利の方向性は DV01 で相殺し、残差の縮小と保有期間のキャリー（銘柄利回り − レポ調達コスト）を取りにいきます。バタフライはレベルとスロープを消して曲率だけを取り、先物ベーシスは現物と先物の裁定益を抜きます。これらは薄いエッジのため、レバレッジをかける前にバックテストで「本当に稼げるシグナルか」をふるいにかける工程が収益を守る要になります。セルサイドのマーケットメイクでも、先物ベーシスと CTD は値付けと在庫管理の基準として使われます。

## 使う bondlab モジュール

| モジュール | 用途 |
|---|---|
| `bondlab.curve`（`fit_nss` / `nss`） | rich/cheap のフィットカーブと残差の算出 |
| `bondlab.curve`（`bootstrap_par` / `DiscountCurve`） | パーカーブからゼロ・フォワードを構築し、キャリー＋ロールを計算 |
| `bondlab.analytics`（`pca` / `duration_convexity`） | バタフライの PCA ウェイト、DV01・コンベクシティによる PnL 分解 |
| `bondlab.bond`（`FixedRateBond`） | 受渡適格銘柄の価格評価とキャッシュフロー生成 |
| `bondlab.bt`（`conversion_factor`） | 先物の CF 計算とベーシス分析 |
| `bondlab.bt`（backtest / performance） | 翌営業日約定のバックテスト実行と評価指標の算出 |

## 検証方針

- **PnL 分解の説明率**：バタフライ等の実 PnL と、キャリー・ロール・カーブ変化・コンベクシティへの分解の合計が 95% 超で一致することを確認する
- **恒等式の数値検証**：キャリー＋ロール ≒ スポット − フォワードを割引カーブ上で突き合わせる
- **ルックアヘッド検査**：lag=0 と lag=1 の成績差を数値で観測し、先読みが混入すると結果が別物になることを可視化する
- **コスト感応度**：取引コストを回転率に比例させて組み込み、コスト後にエッジが残るかを確認する
- バックテスターは翌営業日約定を徹底し、ポイントインタイム性を担保する
