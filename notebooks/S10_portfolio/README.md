# S10 ポートフォリオ

## シリーズ概要

年金・保険・運用会社といったバイサイド（買い手）が、顧客資産をどう組み、
負債やベンチマークに対してどうリスクを合わせ、制約の中でどう超過リターンを
狙うかを扱うシリーズです。個々の銘柄の値付け（S2・S5・S6）やリスク計測
（S3）を土台に、それらを**ポートフォリオ全体の設計問題**へ組み上げます。

扱うのは大きく3つの運用スタイルです。負債を守る ALM／免疫化（保険・年金）、
ベンチマークを少数銘柄で再現するパッシブ複製、制約付き最適化でアルファと
コストを管理するアクティブ運用。いずれも「合わせたいものへエクスポージャを
一致させ、余ったリスクを最小化する」という共通構造を持ちます。

## 学ぶこと

- サープラス $S=A-L$ のテイラー展開から免疫化（Redington）の3条件を導き、
  平行シフトに対する非負サープラスを線形計画で構築する
- 単一デュレーション免疫化がカーブのツイストで破れる仕組みを、キーレート
  デュレーション（KRD）で分解して診断する
- 債券インデックスを完全法で追えない理由（流動性・銘柄数・最低ロット）を
  押さえ、層化サンプリング・セル法・KRD マッチングで少数銘柄複製を設計する
- トラッキングエラーをファクター残差と個別リスクへ分解し、インサンプルと
  アウトオブサンプルで測り分ける
- 平均分散法を債券に当てる難しさ（分布の非対称性・推定誤差の増幅・共分散の
  悪条件）を理解し、ファクターリスクモデルで置き換える
- 取引コスト・回転率制約を凸計画で表現し、バインド制約とシャドープライス
  （双対変数）から最適解を経済的に読み解く

## 実務での使われ方

| スタイル | 主体 | 合わせる対象 | 稼ぎ方 |
|---|---|---|---|
| ALM／免疫化 | 保険・年金 | 負債キャッシュフロー | 積立不足リスクの抑制（顧客への提供価値） |
| パッシブ複製 | インデックスファンド運用 | ベンチマークのリスク特性 | 低コスト・低トラッキングエラー |
| アクティブ最適化 | アクティブ債券ファンド | ベンチマーク＋見立て | 超過リターン（アルファ）とコスト管理 |

免疫化は「負債を守る」、複製は「ベンチマークに合わせる」、最適化は「制約の
中でリターンを最大化する」問題で、どれも運用会社が顧客に約束したリスク特性
を再現・管理する技術です。詳細は `docs/債券ファンドの業務.md` の
「運用会社・ALM」を参照してください。

## 前提

- S2 カーブ構築（ゼロレート・割引率）
- S3-3 キーレートデュレーション、S3-4 主成分（水準・傾き・曲率）
- 修正デュレーション・コンベクシティ・DV01 の基礎（S1）
- 線形計画・凸計画の基本（`scipy.optimize` / `cvxpy`）

## 使う bondlab モジュール

- `bondlab.analytics.duration_convexity` — デュレーション・コンベクシティの算出
- `bondlab.analytics.bump_curve` — カーブのバンプによる KRD 計算
- 外部: `scipy.optimize.linprog`（免疫化の線形計画）、`cvxpy`（複製・最適化の凸計画）

## 検証方針

- **免疫化**: カーブを ±100bp 平行シフトさせ、サープラスが非負に保たれる
  （初期サープラス0が悪化しない）ことを数値で確認する。ツイストでは KRD の
  不一致から免疫が破れることも合わせて示す
- **複製**: インサンプルとアウトオブサンプルの両方でトラッキングエラーを測り、
  構造的なファクター整合が過剰適合に強いことを比較する
- **最適化**: 解の最適性を、バインド制約とシャドープライス（双対変数）の
  相補スラック性で検証し、制約を緩めたときの効用改善量が符号・大きさとも
  整合することを確かめる
